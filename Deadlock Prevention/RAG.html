Resource Allocation Graph (RAG) Algorithm

The Resource Allocation Graph (RAG) is a directed graph used in operating systems to model resource allocation and detect deadlocks. It consists of:

  • Processes (circles) - Represent running programs
  • Resources (rectangles) - Represent system resources (with dots for instances)
  • Request Edges (P → R) - Process is waiting for a resource
  • Assignment Edges (R → P) - Resource is allocated to a process

A deadlock occurs when there's a cycle in the graph where all resources in the cycle have only one instance. The RAG algorithm helps visualize these relationships and detect potential deadlocks.
